package main

import "github.com/wuzzpay/core"

// Smart Failover Routing System
func RouteTransaction(ctx *core.Context) {
    router := core.NewSmartRouter()
    
    // Aliran otomatis ke mitra terstabil
    err := router.ForwardToOptimalGateway(ctx, &core.Opts{
        AllowFailover: true,
        MaxLatencyMs:  200,
    })
    
    if err != nil {
        log.Fatalf("Routing Error: %v", err)
    }
}

Frequently Asked Questions

Wuzzpay beroperasi murni sebagai Technical Service Provider (TSP) atau penyedia sistem teknologi infrastruktur perutean transaksi. Berdasarkan regulasi PBI Nomor 23/6/PBI/2021, penyedia teknologi yang tidak menyelenggarakan transfer dana atau menampung likuiditas tidak diwajibkan memegang lisensi PJP (Penyelenggara Jasa Pembayaran). Seluruh pemrosesan dana dan lisensi gerbang pembayaran tetap melekat penuh pada mitra Payment Gateway berlisensi resmi yang terhubung dengan infrastruktur kami.
Karena Wuzzpay menganut arsitektur Netral/Decoupled, kami tidak menampung atau menyentuh dana transaksi Anda sama sekali. Pencairan dana (*settlement*) langsung ditransfer oleh institusi Payment Gateway pilihan Anda (seperti Winpay, Espay, atau iPaymu) langsung menuju rekening bank bisnis Anda sesuai dengan ketentuan skema SLA dan MDR yang akun Anda miliki dengan pihak gateway penampung tersebut.
Seluruh pengiriman data transaksi di dalam jaringan Wuzzpay diwajibkan melalui protokol enkripsi ketat TLS 1.3. Integrasi API dienkripsi end-to-end menggunakan mekanisme validasi tanda tangan digital HMAC-SHA256 berpasangan guna mencegah pemalsuan data (tampering) atau serangan Man-in-the-Middle.
